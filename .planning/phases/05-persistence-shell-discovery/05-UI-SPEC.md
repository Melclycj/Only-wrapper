# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Persistence + Shell Discovery. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase extends the existing Phase-4 sidebar / row / modal / context-menu / collapsed-rail surfaces.** It does NOT introduce a new design system. All tokens below are pulled verbatim from `.planning/DESIGN.md` (the warm "parlour" system) and the already-shipped token set in `src/renderer/terminal.css`. New surfaces map onto existing tokens; nothing is invented.
>
> **New surfaces this phase owns:** (1) Idle / session card (D-04), (2) Start/Restart row + menu control (D-03), (3) shell dropdown (D-05), (4) welcome / empty state (D-10), (5) dormant-vs-live visual language + drag-to-reorder affordance (D-08). Pre-locked decisions D-01..D-13 are LOCKED contract inputs — formalized here, not relitigated.

## Spacing Scale

The shipped CSS uses a 2px-grid (10px rows, 14px header pad, 22px modal pad) rather than a strict 8-point scale. **This phase honors the existing scale to stay visually consistent with Phase 4 — it does NOT re-baseline spacing.** New surfaces use the multiples-of-4 subset below; the inherited 2px-grid exceptions are documented so the checker reads them as intentional continuity, not drift.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/control gaps inside a row (`.row-controls gap: 4px`), field-label gap |
| sm | 8px | Row inner gap, icon↔name gap, modal action gap, sidebar row vertical pad |
| md | 12px | Field group padding, context-menu item pad, edit-field bottom margin (inherited) |
| lg | 16px | Idle-card inner section gap, modal button horizontal pad |
| xl | 24px | Idle-card outer padding, modal-dialog horizontal pad |
| 2xl | 32px | Welcome/empty-state vertical rhythm (heading → body → CTA) |
| 3xl | 48px | Welcome/empty-state outer breathing room (centered in viewport) |

Exceptions (inherited from Phase 4, intentional — DO NOT "fix"): sidebar row pad `8px 10px`; sidebar pad `10px`; identity header pad `8px 14px`; modal pad `22px 24px 18px`; collapsed rail `52px` basis. New code SHOULD prefer the 4-multiple tokens above; matching an adjacent inherited value (e.g. `10px` row pad) is acceptable for continuity.

---

**Canonical scale for the new surfaces this phase owns: 4 sizes, 2 weights.** All Nunito except the mono role (JetBrains Mono). Inherited Phase-4 values that this phase reuses unchanged are documented in the exceptions block below (mirroring the Spacing section) so they read as intentional continuity, not new declarations.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / caption (idle-card config labels, startup-command helper line) | 12px | 700 (bold) for labels · 400 (regular) for captions | 1.3–1.5 |
| Body / control / mono (idle-card values, Start button label, body copy) | 13px | 400 (regular) · 700 on the Start button label | 1.5 |
| Row / input / dropdown / primary CTA | 14px | 400 (regular) · 700 on the CTA | 1.4 |
| Heading (welcome heading, idle-card session name) | 20px | 700 (bold) | 1.2 |

**Canonical sizes: 12 / 13 / 14 / 20 px. Canonical weights: 400 (regular) + 700 (bold).** Every weight used by a new surface is one of these two; every size is one of these four.

**Inherited typography exceptions (Phase 4 shipped — reused as-is, DO NOT re-baseline):** these already exist in `src/renderer/terminal.css` and are NOT new declarations by this phase — they are documented here only so the checker reads them as deliberate continuity (same pattern as the spacing exceptions):
- `.status-badge` **11px / 700** — the idle-card status badge and sidebar-row badges reuse this shipped component verbatim (no new size introduced).
- `.edit-label` / context-menu item labels at **600 weight** — the shell-dropdown field label reuses the shipped `.edit-label`; new idle-card labels use the canonical 700 instead.
- Modal title (`.modal-dialog h2`) **17px / 700** and other Phase-4 chrome sizes (15 / 16 / 18px) — untouched by this phase. New headings use the canonical **20px**, not 17px.

Notes:
- Welcome heading and the idle-card session name share the single **20px / 700** heading token (anchors the empty state and the card's identity).
- Idle-card field values render in **JetBrains Mono 13px / 400** to read as literal paths/commands and reinforce "this is the saved config" (terminal language = not-yet-running).

## Color

The full palette is `:root` in `terminal.css` (DESIGN.md subset). 60/30/10 split below maps the parlour system; oklch values are authoritative.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--surface` `#ffffff` + `--bg-sunk` `oklch(0.955 0.01 85)` | Sidebar, modal/card surfaces, idle-card body, welcome-state surface. Recessed `--bg-sunk` for grouped/inset regions (restart-group, idle-card config block). |
| Secondary (30%) | `--term-bg` `#1e232c` (viewport gutter) + `--ink`/`--ink-soft`/`--ink-faint` ink ramp + `--line`/`--line-soft` borders | Terminal-area background behind the idle card; all text + hairlines/borders that structure the warm chrome. |
| Accent (10%) | status accents from `STATUS_STYLE` (`status-colors.ts`) — **slate `oklch(0.64 0.02 260)` is the dormant/idle accent this phase leans on** | Reserved list below. |
| Destructive | `oklch(0.58 0.16 25)` (red ramp) | Destructive Close confirmation + close-control hover ONLY (unchanged from Phase 3/4). No NEW destructive action introduced this phase. |

Accent reserved for (explicit — never "all interactive elements"):
- **Status dot + status-badge** on each sidebar row and in the identity header (color = `STATUS_STYLE[status].accent`).
- **Collapsed-rail status dot** (`.collapsed-status-dot`).
- **Idle-card status badge** (the dormant session reads slate `oklch(0.64 0.02 260)` "Idle").
- **Start (▶) control accent on hover/focus** — see Interaction Contracts: the primary launch affordance gets the running-blue `oklch(0.62 0.14 248)` focus ring to signal "this starts the session", distinguishing it from the neutral edit/close controls.
- **Input/dropdown focus ring** `oklch(0.62 0.14 248)` (blue) — already shipped on `.edit-input:focus-visible`; the shell `<select>` inherits it.
- **Drag-active row outline** during reorder — the blue focus ring `oklch(0.62 0.14 248)` (consistent with focus affordances), not a new color.

**Dormant vs live language (D-11):** dormant (`not_started`) reads **slate "Idle"** (low-attention, calm). A live `running` row reads **blue "Running"**. The ONLY differentiator between dormant and live is the **status dot/badge color + label** plus an optional **0.85 opacity on the dormant row's icon+name** (a faint dimming) — NOT a different layout, border, or background. This keeps restored rows obviously present-but-asleep without making them look broken or disabled.

## Copywriting Contract

Tone: warm, first-person-friendly, parlour-not-dev-tool (DESIGN.md aesthetic). Sentence case, no exclamation-spam, no jargon.

| Element | Copy |
|---------|------|
| Primary CTA (welcome/empty state, D-10) | **"Create a session"** (button label; verb + noun, matches D-10 wording) |
| Welcome heading (D-10) | **"Your parlour is quiet"** |
| Welcome body (D-10) | **"No sessions yet. Create one to open a terminal in the folder you choose — it'll be waiting here next time you're back."** (states the empty fact + the value-prop + the next step) |
| Idle-card heading region (D-04) | session **name** (from record) + status badge reading **"Idle"** (no separate heading copy; identity IS the heading) |
| Idle-card config labels (D-04) | **"Working directory"**, **"Shell"**, **"Startup command"** — each above its read-only mono value |
| Idle-card startup-command "displayed-not-executed" cue (D-04) | helper line under the startup-command value: **"Saved for reference — not run automatically. Start the session, then launch it yourself."** (rendered in `--ink-faint`; see Interaction Contracts for the visual treatment) |
| Idle-card empty startup-command | placeholder text **"No startup command saved"** in `--ink-faint` italic (when `startupCommand` is empty) |
| Idle-card Start button (D-03/D-04) | **"▶ Start session"** (the in-card launch point) |
| Sidebar/menu Start label (D-03, `not_started`) | row control `aria-label`: **"Start {name}"**; context-menu item: **"Start"**; glyph ▶ |
| Sidebar/menu Restart label (D-03, has run) | row control `aria-label`: **"Restart {name}"**; context-menu item: **"Restart"**; glyph ↻ (unchanged from Phase 4) |
| Shell dropdown empty/loading (D-05) | while discovering: option text **"Finding shells…"** (disabled). If discovery returns only the `$SHELL` fallback, that single entry still shows (never an empty selector — D-05 safety). |
| Error state — corrupt store recovered (discretion) | **No scary UI.** A recovered-from-corrupt store surfaces as the normal welcome/empty state (D-10). No error dialog, no banner — "surface nothing scarier than the empty state" (CONTEXT discretion). |
| Error state — Start failed (shell spawn error) | session status flips to **"Error"** (red badge, existing `STATUS_STYLE.error`); idle card stays visible with an inline line **"Couldn't start — check the shell and working directory, then try Start again."** in the red ramp. |
| Destructive confirmation | **None new this phase.** Existing destructive Close confirm (Phase 3 D-03a) is unchanged: title "Close session", body warns the running process is killed. Drag-reorder, Start, and shell-select are all non-destructive — no confirm. |

## Interaction Contracts

> The 6 surfaces this phase introduces or extends, spec'd prescriptively for the executor.

### 1. Idle / session card (D-04) — `IdleCard.tsx` (NEW)

Rendered in `.terminal-area` (inside `.viewport-stack` region) when the active session is `not_started`. Replaces the live xterm surface until Start.

- **Container:** centered card, `width: min(440px, calc(100% - 48px))`, `background: var(--surface)`, `border: 1px solid var(--line)`, `border-radius: var(--radius)` (18px), `box-shadow: 0 18px 48px oklch(0.255 0.018 264 / 0.28)` (matches `.modal-dialog`), padding `24px`. Sits on the `--term-bg` gutter (so the card floats on the dark terminal background, signalling "the terminal lives here, but it's asleep").
- **Identity region (top):** `renderIcon(icon, name)` at 28px tile + name (**20px/700**, the canonical heading token) + status badge ("Idle", slate dot — reuses the shipped `.status-badge`, inherited 11px/700). Reuse the `.identity-header` markup conventions (icon → name → badge), gap 8px.
- **Config region:** a `--bg-sunk` inset block (`border-radius: 12px`, padding `16px`), three stacked label+value pairs. Labels: **12px/700** `--ink-soft`. Values: **JetBrains Mono 13px/400** `--ink`. Order: Working directory, Shell, Startup command.
- **Displayed-not-executed cue (startup command):** the startup-command value renders in mono with a small leading **"$ "** prompt glyph in `--ink-faint` (reads like a terminal line you'd type) but is **visually inert** — no run button, no copy-as-command affordance. Directly beneath it: the helper line (`--ink-faint`, **12px/400**, line-height 1.5) "Saved for reference — not run automatically…". This is the deliberate TERM-05 boundary cue.
- **Start button (bottom):** full-width-ish primary button "▶ Start session", `background: var(--surface)`, `border: 1px solid oklch(0.62 0.14 248)`, text `oklch(0.62 0.14 248)` (blue = "go live"); hover fills `oklch(0.62 0.14 248)` bg + white text. `border-radius: 10px`, padding `8px 16px`, **13px/700**. Focus-visible: 2px blue outline, offset 2px. This is the in-card launch point (second to the sidebar ▶).
- **State coverage:** loading (n/a — record is local), empty startup-command (placeholder copy), error-after-start (inline red line; card stays).

### 2. Start / Restart control (D-03) — extends existing `.row-controls` + `ContextMenu`

The Phase-4 `.row-controls` slot ALREADY renders a Restart (↻) control for non-running rows. Phase 5 generalizes that single control into a **Start/Restart** control:

- **Label flip (D-03):** `not_started` → glyph **▶**, `aria-label="Start {name}"`, `data-action="start"`, `data-testid="start-session"`. Once the session has run (`running`/`stopped`/`exited`/`error`) → glyph **↻**, `aria-label="Restart {name}"`, `data-action="restart"`, `data-testid="restart-session"` (unchanged). `running` rows show ↻ Restart (existing behavior).
- **Affordance:** reuse the existing `.row-control` square icon button (24×24, transparent border, `--ink-soft`, rounds + warms on hover to `--bg-sunk`/`--ink`). It lives in the same `.row-controls` cluster, revealed on row hover/active (opacity 0→1), to the LEFT of Edit ✎ and Close ✕.
- **Start-specific accent:** the ▶ (Start) control on a `not_started` row gets a **blue hover/focus** treatment (border + text → `oklch(0.62 0.14 248)`) to read as the primary "go" affordance, distinct from the neutral Edit and the red Close. Add a `.row-control-start:hover` rule mirroring the existing `.row-control-close:hover` pattern but using the blue ramp. Focus-visible: 2px blue outline.
- **States:** hover (warm/blue), focus-visible (blue outline), disabled (none — Start/Restart are always available for a selectable row; a `running` row simply shows Restart). Click `stopPropagation` (does not trigger row-switch), fires `onStart`/`onRestart`.
- **Context-menu parity (D-03):** the `ContextMenu` "Restart" item becomes "Start" when `not_started`, "Restart" otherwise — same label logic, reusing `.context-menu-item`. This is the collapsed-rail control surface (where row buttons are hidden), per Phase-4 D-11.
- **Clicking a dormant row body = select/view only** (shows the idle card). Start is the explicit ▶ / menu action — never lazy-spawn on row click (D-03).

### 3. Shell dropdown (D-05) — replaces free-text shell field in `SessionEditModal.tsx`

- **Control:** a native `<select>` styled to match `.edit-input` exactly (box-sizing border-box, full width, padding `8px 10px`, `border: 1px solid var(--line)`, `border-radius: 10px`, `--surface` bg, `--ink` text, Nunito 14px). Add a `.edit-select` class (or extend `.edit-input` to cover `select`) with the SAME `:focus-visible` blue outline. A subtle custom caret (▾) is acceptable but optional — native select chrome is fine.
- **Population:** options come from `discoverShells()` IPC. Each option: `label` = basename (e.g. "zsh", "bash"), `value` = full path. Resolved `$SHELL` is ALWAYS present and is the **default-selected** option (D-05 safety).
- **States:** while the IPC is in flight, show a single disabled option "Finding shells…"; on resolve, replace with the discovered list. If discovery yields only `$SHELL`, that one option shows (selector never empty). The field lives in the existing `.edit-restart-group` ("Applies on restart") — shell changes take effect on next Start/Restart (Phase-4 D-02 unchanged).
- **No free-text (D-05):** the editable path input is REMOVED. Users cannot type an arbitrary path (this also removes the renderer's ability to inject an arbitrary executable — security V5).

### 4. Welcome / empty state (D-10) — `WelcomeEmptyState.tsx` (NEW)

Shown when `sessions.length === 0` (first-ever launch OR after closing every session). Replaces the old auto-add-default boot behavior.

- **Layout:** fills the whole app area (no sidebar rows to show, but the sidebar chrome + collapse toggle may remain). Content centered both axes, on `--bg-sunk` (warm, not the dark terminal gutter — this is a "no terminal yet" state). Max content width ~360px.
- **Composition (vertical, gap 32px rhythm):** an oversized friendly glyph (🛋️ — the parlour motif, ~48px) → heading "Your parlour is quiet" (20px/700 `--ink`, line-height 1.2) → body copy (13px/400 `--ink-soft`, line-height 1.5, centered) → primary CTA button.
- **CTA treatment:** "Create a session" — the ONE prominent button in this state. `background: oklch(0.62 0.14 248)` (blue, filled — the single high-emphasis action), white text, `border-radius: 10px`, padding `10px 20px`, 14px/700. Hover darkens to `oklch(0.58 0.14 248)`. Focus-visible: 2px blue outline offset 2px. `data-testid="welcome-create-session"`.
- **Behavior:** clicking it runs the existing quick-add path (D-11: new = live, spawns immediately). Nothing auto-spawns on boot.

### 5. Dormant-vs-live language + drag-to-reorder (D-08, D-11)

- **Dormant row reading:** see Color §"Dormant vs live language" — slate "Idle" badge + 0.85 icon/name opacity. No disabled styling, no separate border.
- **Drag handle:** drag-to-reorder uses `@dnd-kit/sortable` (RESEARCH A1). The whole row is the drag surface via a pointer sensor with an activation distance (so a click still switches and a click on a control still fires); a dedicated **drag handle glyph (⠿, U+2820 / "⋮⋮")** appears at the row's left on hover in `--ink-faint`, `cursor: grab` (→ `grabbing` while dragging), to telegraph draggability. Keyboard reorder is supported by dnd-kit (a11y).
- **Dragging state:** the lifted row gets `opacity: 0.9`, the blue focus ring outline (`oklch(0.62 0.14 248)`), and a soft shadow `0 6px 18px oklch(0.32 0.012 70 / 0.14)` (matches the rail-tooltip shadow). The drop target gap is indicated by dnd-kit's transform; no custom insertion line required for MVP.
- **Collapsed-rail DnD (D-08 discretion):** expanded-mode dragging is required; full collapsed-rail DnD is optional. The collapsed rail MUST reflect the saved order (render order = `order`).
- **Persistence is invisible (D-13):** reordering, starting, editing all persist via debounced write — there is NO save button, spinner, or "saved" toast. Persistence is silent by contract.
