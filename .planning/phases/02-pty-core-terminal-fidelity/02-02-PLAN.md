<objective>
Build the main-process half of the PTY round-trip: the real resolveShell() (login shell, native PATH), the flow-control watermark accounting, and the PtyManager that owns a node-pty process keyed by LogicalId — exposing validated, security-clamped IPC handlers (create/write/resize/pause/resume) and main-to-renderer data/exit events, with lifecycle cleanup. It also extends the typed bridge contract and the EXPECTED_API_KEYS guard so the renderer (02-03) can consume the surface.

Purpose: This is the producer side of the Core Value. Without a correctly-spawned login PTY with native PATH and inherited env, `claude --rc`/`codex` will not resolve (TERM-03); without validated IPC, the bridge is an EoP/DoS surface. After this plan, the main process can spawn, stream, resize, pause/resume, and cleanly kill a real PTY — but nothing renders yet (that is 02-03).
Output: shell-resolver.ts (real), flow-control.ts, pty-manager.ts, extended api-types.ts + window-config.ts EXPECTED_API_KEYS, wired index.ts, and a new IPC-validation unit test. Turns the shell-resolver and flow-control RED unit tests GREEN; updates the security guard test contract.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (contextBridge IPC) | Untrusted renderer-supplied args (id, cols, rows, data) cross into the main process and reach a native PTY |
| PTY child ↔ host env | The spawned shell inherits the full parent env (intentional, D-01) and runs with the user's own privileges |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Elevation of Privilege | pty:write / pty:create IPC | mitigate | The PTY only receives user-typed keystroke bytes; main never eval/exec renderer strings as commands. contextIsolation+sandbox preserved; only the 7 narrow typed methods are exposed. write data must be a string (type guard) before reaching pty.write. |
| T-02-03 | Denial of Service | pty:resize (cols/rows) | mitigate | clampDimension clamps cols/rows to 1-1000 (NaN/0 → 1) before pty.resize — defeats the resize-bomb DoS. |
| T-02-04 | Tampering | pty:write/resize/pause/resume id arg | mitigate | Every handler validates id against the live PtyManager map; unknown/forged ids are ignored (single id this phase, enforced now for Phase-3 multi-session). |
| T-02-05 | Information Disclosure | PTY output logging | mitigate | Never log raw PTY byte streams (may contain secrets/keystrokes); log lifecycle events only (spawn pid, exit code) — Security V7. |
| T-02-06 | Denial of Service | orphaned shell child processes | mitigate | disposeAll()/pty.kill() on window 'closed' and app 'before-quit' (Pitfall 6). |
| T-02-07 | Tampering | EXPECTED_API_KEYS expansion | accept | Adding the 7 PTY keys deliberately loosens the Phase-1 surface; the security.guard.test.ts still enforces the EXACT new set so no unreviewed key can leak. Reviewed, intentional. |
</threat_model>
