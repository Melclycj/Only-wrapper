<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main → renderer (pty:data / pty:exit events) | Main-process PTY output crosses into the renderer; the renderer must not be handed the raw IpcRendererEvent or raw ipcRenderer |
| renderer DOM → window.api | All renderer access to the PTY is via the narrow typed bridge; electron/node-pty imports are ESLint-banned in renderer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Elevation of Privilege | preload bridge surface | mitigate | Exactly one contextBridge.exposeInMainWorld('api', ...); no raw ipcRenderer; onPtyData/onPtyExit forward only the typed payload field, never the IpcRendererEvent. security.guard.test.ts enforces the exact key set. |
| T-02-09 | Information Disclosure | PTY output in renderer logs | mitigate | The renderer writes PTY data straight to xterm and does not console.log raw stream content (Security V7 — applies to renderer too). |
| T-02-10 | Tampering | .node loaded from ASAR/writable path | mitigate | auto-unpack-natives (already in forge.config.ts) unpacks node-pty .node to the read-only resources dir; the packaging smoke in Task 3 verifies the packaged app boots and round-trips (Pitfall 4 / Security V12). |
