producer AND verifier of the cross-platform artifacts (D-01) — dev/test is macOS-only, so CI

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → committed repo | the workflow must build with ZERO secrets — no signing credential is ever referenced or required this phase |
| CI artifact → user | the uploaded .exe/.app is UNSIGNED this phase; the user is told via docs/PACKAGING.md (local-open path, signing is a later env-gated flip) |
| OS host → app boot | the live SC4 check confirms the pre-1809 native dialog gate fires before any node-pty spawn |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-09 | Information Disclosure | CI workflow secrets | mitigate | The workflow references ZERO secrets.* and requires no signing creds (negative grep enforces); unsigned build is the D-04 lock; CI needs no secrets this phase |
| T-08-10 | Tampering | maker-squirrel auto-sign hang | mitigate | Leave windowsSign UNSET (Pitfall 4 — a stray cert config hangs CI); the negative grep on windowsSign enforces it; unsigned Squirrel builds without signtool |
| T-08-11 | Denial of Service / Cost | CI matrix runner spend / retry storm | accept | Two short legs (npm ci + make + one smoke run) per push/PR; fail-fast:false isolates a Windows flake from the macOS leg; smoke is best-effort if headless-flaky so a flaky runner does not trigger endless re-runs |
| T-08-12 | Denial of Service (self-inflicted) | pre-1809 Windows live boot | mitigate | The live SC4 human-check confirms the native dialog + clean quit (the os-gate logic is already unit-proven in Plan 01) |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | This plan installs ZERO new packages; CI runs `npm ci` against the locked, exact-pinned lockfile (no new package, no slopcheck gate needed) |

## Reliability & Cost Lens
- make = hard gate, smoke = strong-preferred (best-effort on headless flake), human-verify = binding SC2 gate — this layering prevents a flaky CI runner from blocking the phase while keeping the real artifact-build a hard requirement.
- fail-fast:false + a single smoke run per leg bound the runner cost; no retry loop, no scheduled re-trigger.
</threat_model>
