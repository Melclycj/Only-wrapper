**Deterministic testing notes (from research):**
- **Pure-unit-testable (no real PTY):** marker build, `matches()` matcher against captured byte fixtures, the timeout state machine (settle vs flush-on-timeout), the non-empty gate. Reuse the existing `node-pty`-mock harness (`FakeChild._fireData`/`_fireExit`, as in `pty-lifecycle.test.ts`) to fire synthetic marker bytes through mocked `onData` and assert: (a) probe bytes are NOT sent to the renderer, (b) on match the real command is written with `\r`, (c) on timeout the buffer is flushed and no command is written.
- **Requires real cold spawn (E2E only):** the actual "no garble" + "lands in history" guarantees — drive via WDIO against the built app; create a session with a startup command, assert the command output appears AND the nonce does NOT appear in the buffer, then assert `history` / up-arrow recalls it. Cold spawn = a freshly created session (first spawn) via existing `ensureSession` / `xterm-driver` helpers.
