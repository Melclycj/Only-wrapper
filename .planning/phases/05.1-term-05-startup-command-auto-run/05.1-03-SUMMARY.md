## Decisions Made
- **Timeout duration (Open Q2):** `READINESS_TIMEOUT_MS = 4000` ms — real cold zsh settles the `: <nonce>` round-trip in ~seconds, comfortably under the 8s smoke `waitForText`; no tuning needed.
- **Notice transport (Open Q1):** reuse `onPtyStatus` via optional `PtyStatusPayload.notice` — zero new bridge keys, the lighter path.
- **Matcher vs real bytes (Open Q3):** `buildPosixProbe` regex needed NO change against real cold zsh; a defensive `stripProbeEcho` scrub was added to the post-settle forwarding path so the `__JW_READY_` sentinel can never leak under adversarial chunk timing.
- **Security (V7 / T-05.1-04):** the notice is a fixed literal — it interpolates no `startupCommand`, nonce, or buffered bytes.
