# Phase 05.1 Plan 01: ReadinessProbe Seam + Wave 0 Scaffold Summary

Interface-first foundation for TERM-05 startup-command auto-run: a pure, electron/node-pty-free `ReadinessProbe` seam (POSIX `:` no-op probe + send-vs-match matcher mirroring `shell-discovery.ts`) plus the full Wave 0 RED test scaffold pinning SC1–SC5 / D-02 / D-04 before any behavior lands.

## What Was Built

**Task 1 — `src/main/readiness-probe.ts` (the seam):**
- `ShellReadinessProbe` (`marker: string`, `matches(buffer): boolean`) + `ReadinessProbeProvider` (`forShell(shellPath)`) interfaces.
- Pure `buildPosixProbe(nonce)`: marker `: ${nonce}\r` (POSIX `:` no-op builtin + CR terminator per Pitfall 4 / ICRNL; D-01 — changes no shell state); `matches` returns true only when the nonce appears on a produced line FOLLOWED by a newline (send-vs-match split, Pitfall 1), false on the bare echoed-input line. Nonce regex metacharacters escaped; matcher documented E2E-tunable.
- `MacReadinessProbe` (zsh + bash share the POSIX probe — one covers both; `__JW_READY_<hex>__` crypto-nonce sentinel).
- `WindowsReadinessProbe` stub whose `forShell()` THROWS (Phase-8, D-03).
- `selectReadinessProbe(platform)` picker (`win32` → Windows stub; else Mac).
- Electron-free and node-pty-free (grep-verified); type-checks + lints clean.

**Task 2 — Wave 0 RED scaffolds:**
- `src/main/__tests__/readiness-probe.test.ts`: pure-helper group (marker shape, send-vs-match `matches`, Windows-throws, Mac picker) GREEN; FakeChild state-machine group RED until Plans 02/03 implement the create() probe gate — asserts probe-byte withholding (D-02), inject-on-match `cmd + "\r"` (SC1), flush-on-timeout-no-inject (D-04/SC4), empty-command no-inject (SC2).
- `tests/smoke/startup-command.smoke.test.ts`: cold-spawn E2E scaffold — SC1 (output `JW_STARTUP_OK` visible + command typed + ArrowUp history), D-02 (`__JW_READY_` absent), SC3 (restart re-run after `restarted` separator), SC4/D-04 (timeout-fallback documented sketch).
- `src/main/__tests__/pty-hydrate.test.ts`: SC5 extension — `hydrate([record with startupCommand])` spawns no PTY → never auto-runs (D-06); stays GREEN.
