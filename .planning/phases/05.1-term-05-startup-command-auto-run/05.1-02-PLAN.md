<objective>
Deliver the core vertical slice of TERM-05: after a PTY spawns with a non-empty `startupCommand`, run the invisible one-shot readiness probe and — once the shell genuinely accepts input — inject the command so it runs and lands in shell history. This is the thinnest end-to-end path that gives the user the capability: ▶ Start a session carrying `claude --rc` and the tool relaunches in its project dir without retyping (SC1). The happy-path probe (probe → ready → inject) lands here; the timeout/notice fallback (D-04) and the E2E/restart verification land in Plan 03.

Per D-05, ONE hook at the existing `child.onData` wiring in `create()` (line ~281) covers all three trigger surfaces (new-with-command, Restart-respawn, dormant Start-promotion) because they all converge on `create()`. Per D-02, invisibility is achieved by the load-bearing architectural lever: the probe phase installs a transient `onData` interceptor that BUFFERS bytes and never calls `this.send()`; on ready it discards the probe bytes and restores normal forwarding via the shared `wireNormalOnData()`. Per the locked injection model, the command is written main-side as `cmd + '\r'` (CR — a real Enter; bypasses the renderer bracketed-paste path that exists to PREVENT auto-execute), so it echoes visibly and lands in history (SC1).

Purpose: Replace guessing-by-delay (the twice-descoped cold-spawn garble bug) with confirming-readiness. Output: `wireNormalOnData()` extracted as a shared forwarder + the probe-then-inject hook in `create()`, turning the Plan 01 probe-hook unit assertions GREEN for the match/inject/withhold/empty-gate cases.
</objective>
