### `src/main/pty-manager.ts` (MODIFIED ‚Äî service, the single spawn owner)

**Analog:** itself. The modification lands at the existing `child.onData` wiring inside `create()`.

**Current onData wiring to refactor** (lines 279-283):
```typescript
// Forward the UTF-8 string straight through ‚Äî no binary re-encoding ...
child.onData((data) => {
  this.send(PTY_CHANNELS.data, { id, data });
});
```
RESEARCH (line 156) prescribes refactoring this into a shared private method `wireNormalOnData(id, child)` so both the normal path and the post-probe resume share one definition. The probe-then-inject hook (RESEARCH lines 122-155) replaces this block, gated on `record.startupCommand?.trim()`.

**The `send()` invisibility lever** (lines 393-398) ‚Äî the load-bearing fact:
```typescript
private send(channel: string, payload: unknown): void {
  const w = this.win;
  if (w && !w.isDestroyed() && !w.webContents.isDestroyed()) {
    w.webContents.send(channel, payload);
  }
}
```
Probe invisibility (D-02) = the transient onData interceptor BUFFERS bytes and simply never calls `this.send(PTY_CHANNELS.data, ‚Ä¶)` for them. On ready: discard the buffer, `wireNormalOnData`, then `child.write(cmd + '\r')`. On timeout: `this.send(PTY_CHANNELS.data, { id, data: buffer })` to flush the bare prompt (SC4), then emit the D-04 notice.

**Injection write convention** ‚Äî use `child.write(cmd + '\r')` (CR, not LF). The codebase + POSIX termios ICRNL basis is RESEARCH lines 159-167; the existing `FakeChild.write` is a `vi.fn()` so unit tests assert the exact `'\r'`-terminated argument.

**Channel/constant conventions:**
- Reference channels via the `PTY_CHANNELS` object (lines 33-55); if a new readyFail channel is added, add it there (e.g. `readyFail: 'pty:ready-fail'`) ‚Äî but prefer reusing `PTY_CHANNELS.status` per Open Q1.
- Add a named timeout constant `READINESS_TIMEOUT_MS` mirroring the existing `STOP_GRACE_MS` style (RESEARCH Open Q2; ~4000 ms).
- Lifecycle logging only ‚Äî NEVER log the command or PTY bytes (Security V7, mirrors line 269/270 `console.log('[pty] spawned ‚Ä¶')`).

**Restart-separator ordering pitfall (A6 / Pitfall 2):** the `‚Äî restarted HH:MM ‚Äî` separator is written **renderer-side** in `SessionView.tsx:219` (`term.write(...)`), NOT in main. It therefore does NOT pass through the main-side probe buffer and cannot be swallowed by the probe. Confirm this during planning ‚Äî it resolves the RESEARCH ordering concern favorably (the separator is already independent of the probe stream).

**Dormant SC5 (D-06):** no code ‚Äî `restart()` (lines 441-471) and the dormant promotion path both funnel through `create()`; restored-but-not-started records live in `dormantRecords` with no PTY, so nothing to inject into.

## Shared Patterns

### Platform seam (macOS-now / Windows-Phase-8)

Interface + pure helpers + Mac provider (implemented) + Windows provider (stub) + `select‚Ä¶(platform)` picker. Windows stub THROWS this phase (RESEARCH line 279) rather than returning a default, since there is no safe no-op probe.

### Main-owns-the-byte-stream (invisibility + injection)
**Source:** `src/main/pty-manager.ts` lines 281-283 (onData‚Üísend) + 393-398 (send guard) + `FakeChild.write`
**Apply to:** the probe hook in `create()`
Invisibility = withhold bytes from `send()`; injection = main-side `child.write(cmd + '\r')`. Renderer NEVER gets a probe byte and has no auto-run write path (renderer `ptyWrite` is the user-typing path only).

**Source:** `src/main/__tests__/pty-lifecycle.test.ts` lines 40-104
**Apply to:** `src/main/__tests__/readiness-probe.test.ts`
`vi.mock('node-pty')` + `FakeChild` with `_fireData`/`_fireExit`, `vi.useFakeTimers()` for the timeout state machine, `fakeWindow().webContents.send` spy for the withhold/flush assertions.

### xterm-driver E2E smoke

**Apply to:** the probe + injection code. Log lifecycle, never the nonce, the buffered bytes, or `startupCommand`.
