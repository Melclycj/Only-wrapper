## Process states & Start surface (Axis 1)

Discussed the stop/start model. Initial framing offered an explicit Stop⇄Start
toggle vs a destructive-Close-only model. The user reframed the whole model around
**reducing steps** and a two-bucket lifecycle, which is captured below.

| Option (stop/start) | Description | Selected |
|--------|-------------|----------|
| Explicit Stop ⇄ Start toggle | Running session has Stop (keep as 'stopped'); Start revives | |
| No Stop button; exit + prominent Start | Destructive-Close-only; rely on exit | |
| Something else (user's two-bucket model) | Working Area (live) + Inactive List (dormant configured); no Stop verb | ✓ |

**User's choice:** Two-bucket model. *"Reduce the steps for user to achieve simple
tasks such as start a terminal."* Ephemeral sessions (no metadata) are throwaway and
not persisted; configured sessions (any metadata) persist and move to an Inactive List
on Remove, with a Start button to revive and a Delete to purge.

Follow-ups:
- **Ephemeral → configured trigger:** *auto-save the moment any metadata is given.*
- **What persists when inactive:** the user asked metadata vs the exact session.
  Resolved: you cannot persist a live process across restart, so only the **recipe**
  persists. Chose **no history on Start** (clean fresh start) — *"Agent resume will do
  the work of previous context. For the app, start with no history."*
- **Process exits on its own:** **(a)** drops straight to the Inactive List (Working
  Area = live only).
- **App restart restore:** **yes** — configured sessions return in the Inactive List;
  first Start shows no `— restarted —` separator (fixes that bug).

**Notes:** Confirmed a live session does consume resources (process + buffer + GPU
context), which justifies an Inactive List with no live process over backgrounding
everything.

## Restart & restore behavior (Axis 1)

Largely resolved within the lifecycle discussion.

**Decisions:** In-place Restart preserves scrollback + writes the `— restarted —`
separator + exits alt-screen first (and must stop trimming on repeat); app-restart
restore lands in the Inactive List with no spurious separator on first Start.
