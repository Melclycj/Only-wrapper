### DEFECT A — Remove of a live configured session reverts Inactive→Working

- **Chain:** configured-live Remove (`SessionManager.confirmClose`) → `window.api.ptyStop` →
  main `stop()` set `userStopped=true` → onExit `deriveStatus` → `'stopped'`; the record
  stayed live as `'stopped'` (self-exit routing was skipped because `selfExit` excludes
  userStopped). Renderer `resolveRowStatus` only mapped `'exited'`/`'error'` → not_started, so
  `'stopped'` passed through and the Sidebar partition put the row back in the Working Area.
- **Fix (main, the chosen mechanism):** new `PtyManager.removeLive()` — the configured-live
  Remove primitive. It kills the PTY like `stop()` but marks the session `removing`, so onExit
  (a) routes the kept record to `dormantRecords` as `not_started` (order preserved) AND
  (b) BROADCASTS `'not_started'` (the record's real terminal state) instead of `'stopped'`.
  The IPC `pty:stop` handler now routes to `removeLive()` (the renderer's only stop-a-live
  gesture — the Stop verb was abolished, D-01). `stop()` is UNCHANGED (the restart precursor —
  still keeps the record live as `'stopped'` so `restart()` can respawn).
- **Why NOT generalize `'stopped'`→not_started renderer-side:** that was the first attempt and
  it broke a RESTART's transient stopped→running — it unmounted the kept SessionView mid-restart,
  dropping the `'— restarted —'` separator and the startup-command re-run (smoke SC3 went RED).
  Reverted; the Remove is handled main-side as a `'not_started'` broadcast instead. `'stopped'`
  now passes through the renderer reducer untouched (restart safety).
- **restart-in-place verified ALIVE:** `restart()` is still wired (context-menu "Restart" +
  sidebar ↻ + `handleRestart`). It calls `this.stop(id)` internally (NOT via IPC), so it keeps
  the live-`'stopped'`→respawn path. The IPC reroute to `removeLive()` does not touch it.
- **Locks (RED→GREEN):** `pty-lifecycle.test.ts` removeLive()-lands-dormant test; the old
  `stop()`-stays-`'stopped'` contract realigned (stop() = restart precursor only) + comment.
  `session-status.test.ts` / `apply-status-event.test.ts`: `'stopped'` passes through; the
  `'not_started'` Remove broadcast lands the row dormant + drops pid/overlay (`movedToInactive`
  now fires on ANY resolution to not_started from a previously-live row).

### DEFECT B — dev close+restart loses the session (reboot keeps it)

- **Root cause:** dev + packaged share ONE store file; on macOS `window-all-closed` only quits
  on non-darwin, so closing the dev window fires no `before-quit`; the old `win.on('closed')`
  ran `disposeAll()` WITHOUT flushing the store. A session created within the ~300ms debounce
  was lost. On-disk file was stuck at `version:1` (SCHEMA_VERSION is 2) — writes re-persisted
  the stale version because lowdb `read()` replaces `db.data` wholesale.
- **Fix:** new `src/main/lifecycle.ts` (electron-free, unit-testable): `handleWindowClosed`
  flushes the store BEFORE `disposeAll()`; `handleWindowAllClosed` flushes on ALL platforms
