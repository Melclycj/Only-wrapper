<objective>
Axis 1 (TERM-12 lifecycle) main-side: implement the two-bucket persistence/transition model
that the renderer UI (Plan 04) presents. `updateProfile()` auto-promotes a session to
`configured` on any metadata edit (D-02). `onExit` routes a configured self-exit to the
Inactive List (move to `dormantRecords` as `not_started`) and an ephemeral self-exit to gone
(D-05), while a spawn-failure stays an error row (retained). The store persists only
configured records. This turns GREEN the three RED lifecycle scaffolds Plan 01 created.

Purpose: The Inactive List IS the dormant-restore model surfaced as UI; "stopped session =
a recipe, not a frozen process." Decoupling persistence (configured-only) from the live map
gives the renderer a clean "live `sessions` only" Working-Area filter.
Output: A refactored `pty-manager.ts` (updateProfile + onExit + `listConfiguredSessions()`),
`index.ts` snapshot filter, and the Plan-01 lifecycle scaffolds GREEN.
</objective>
