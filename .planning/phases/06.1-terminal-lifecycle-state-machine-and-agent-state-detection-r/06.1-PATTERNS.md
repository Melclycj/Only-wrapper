**Analog:** its own `classifyIdle`/`PROMPT_RE` (the WRONG model) → replace with the spike `record.cjs` `classify()`.

**CURRENT — the exact lines to remove (D-10 kill-finding: `❯` must go):**
```ts

// agent-state.ts l.49  — PROMPT_RE STILL contains ❯ (the 002 false-fire)
```

The `AgentState` type (l.19 `'in-progress' | 'waiting' | 'free'`) is CORRECT and stays.

**REPLACE WITH — `record.cjs` `classify()` (l.67–95), the 002-corrected form:**
```ts
// Source: .planning/spikes/001-frame-stability-mechanism/record.cjs l.67–95 (verbatim shape)
function classify(lines: string[]): AgentState {
  const nonEmpty = lines.filter((l) => l.trim() !== '');                      // 001 finding #1/#3
  const last = nonEmpty.length ? nonEmpty[nonEmpty.length - 1] : '';
  const region = nonEmpty.slice(-4).join('\n');                              // 001 finding #2 (tight window)
  const shellPrompt = /[$%#]\s*$/.test(last) || /\w[^>]>\s*$/.test(last);
  const trailingQuestion = /\?\s*$/.test(last);
  const ynBracket = /\[y\/n\]|\(y\/n\)|\(yes\/no\)/i.test(region);
  const numberedMenu = (region.match(/^\s*[❯>]?\s*\d+\.\s+\S/gm) || []).length >= 2;
  const claudeFooter = /(esc to (cancel|interrupt)|tab to amend|ctrl\+e to|↑↓ to|enter to)/i.test(region);
  const passwordPrompt = /(password|passphrase).*:\s*$/i.test(last);
  // ❯ (arrow_marker) DELIBERATELY ABSENT from the decision (002 kill-finding).
  if (shellPrompt && !trailingQuestion) return 'free';                       // authoritative FREE
  if (numberedMenu || ynBracket || trailingQuestion || passwordPrompt || claudeFooter) return 'waiting';
  return 'free';
}
```
`lastNonEmptyLine` (l.58–66) + `ANSI_RE` (l.33–34) can stay as helpers, but the new path
reads `translateToString(true)` lines (already ANSI-interpreted) so ANSI-strip may be
unnecessary on the viewport input. Keep the purity header + named-constant + documented-
contract convention. All regexes anchored/linear → ReDoS-safe on a `rows×cols`-bounded viewport.

**REPLACE WITH — a `setInterval` tick over the live buffer (`record.cjs` viewportLines/fnv1a, RESEARCH Pattern 1):**
```ts
// Source: record.cjs l.46–60 + the tick loop l.116–140, adapted to the LIVE Terminal
const TICK_MS = 100;             // discretion ~100–250ms — do NOT exceed ~250ms (Nyquist)
const SETTLE_MS = 500;           // discretion 400–600ms
let lastHash: string | null = null, changeAt = performance.now();
const viewportLines = (): string[] => {
  const b = term.buffer.active; const top = b.viewportY; const out: string[] = [];
  for (let i = 0; i < term.rows; i++) { const ln = b.getLine(top + i); out.push(ln ? ln.translateToString(true) : ''); }
  return out;
};
const fnv1a = (s: string): string => { let h = 0x811c9dc5; for (let i=0;i<s.length;i++){h^=s.charCodeAt(i);h=Math.imul(h,0x01000193);} return (h>>>0).toString(16); };
const agentTick = setInterval(() => {
  if (!agentRunning) return;                       // gate (set true on 'running', false on exit) — KEEP
  const lines = viewportLines(); const h = fnv1a(lines.join('\n'));
  if (h !== lastHash) { lastHash = h; changeAt = performance.now(); emitAgent('in-progress'); }
  else if (performance.now() - changeAt >= SETTLE_MS) { emitAgent(classify(lines)); }
}, TICK_MS);
```
- **KEEP** `emitAgent` change-only (l.235–239) and the `agentRunning` gate (set at l.321 / cleared at l.338).
- **Lifecycle (RESEARCH Pattern 3 / Pitfall 7):** arm in the mount effect (keyed on `id` only — l.391); clear `agentTick` in the cleanup (mirror `idleTimer` clear at l.369); on restart the SAME term persists → just reset `lastHash`/`changeAt`. Hidden panes still tick (term.write runs unconditionally, l.248) — REQUIRED for D-12.

**D-05 self-exit→Inactive — CURRENT `onExit` KEEPS the record live-but-dead (l.479–484):**

**CHANGE (D-05, RESEARCH §D-05 + Open-Q3):** on a *self-exit* (`userStopped===false`, status `exited`/`error`-but-reached-running) of a **configured** session → move the record `this.sessions` → `this.dormantRecords` (coerce `status:'not_started'`) so `listSessions()` renders it as an Inactive-List entry. **`restart` (l.650–690) and `listSessions` (l.699–703) — UNCHANGED structure.** `restart` already kills-then-respawns under the same `logicalId` (IDENT-02). `listSessions` already merges `live ∪ dormant` sorted by `order` — this IS the Inactive List source.

**Source:** `SessionView.tsx` `emitAgent` (l.235–239) + the gate set/clear (l.321 / l.338–344).
**Apply to:** the new frame-stability tick — keep both so the parent never churns and a
stopped/exited session is never classified.

### Frame read = `term.buffer.active`, never scrollback (Don't-Hand-Roll)
**Source:** `record.cjs` `viewportLines` (l.46–55) — `b.viewportY`..`+term.rows`.
**Apply to:** SEAM A. Reading scrollback re-introduces the 001 stale-menu false positive.

### Main is authoritative; renderer reconciles `listSessions()`
**Source:** `pty-manager.ts` `listSessions()` (l.699–703) merging `live ∪ dormant`.
**Apply to:** bucketing — the Working/Inactive partition is a render-time predicate over this
snapshot, NOT a new state machine.

### Notice/frame writes are sanitized literals (V5 / ANSI-injection)
**Source:** `SessionView.tsx` `sanitizeNotice` (l.92–94) + main `sanitizeNotice`.
**Apply to:** the new `MOUSE_RESET`/separator writes are fixed literals; the only interpolated
text (the SC2 notice) is already control-char-stripped both sides.
