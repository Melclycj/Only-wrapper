# Phase 06.1 Plan 03: Two-Bucket Lifecycle (TERM-12 main-side) Summary

**`updateProfile()` one-way auto-promotes a session to `configured` on any metadata edit (D-02), the persisted snapshot filters to `configured===true` via the new `listConfiguredSessions()`, and `onExit` routes a configured self-exit to the Inactive List (dormant `not_started`, order preserved), an ephemeral self-exit to gone, and leaves a user-stopped restart precursor in place (D-05) — turning the three Wave-0 RED lifecycle scaffolds GREEN.**

## Performance

- **Duration:** ~12 min

## Accomplishments

- **Task 1 (D-02):** Added `target.configured = true` after the field writes in `updateProfile()` — ANY metadata write (name/icon/cwd/shell/startupCommand) one-way promotes the session to configured; `create()` is untouched so a bare `+ New` session stays ephemeral. Added `PtyManager.listConfiguredSessions()` filtering `listSessions()` to `configured===true`, and switched `index.ts` `syncStore()` to feed the store from it so an unedited ephemeral session never reaches disk. The Plan-01 "only configured records persist" scaffold is GREEN; `session-store.ts setSessions` stays a dumb setter (untouched).
- **Task 2 (D-05):** Replaced the keep-live-but-dead block in `onExit` with the two-bucket `selfExit` routing. A configured self-exit MOVES the record `this.sessions` → `this.dormantRecords` coerced to `not_started` with the pid dropped and `order` preserved (RESEARCH A2) → it reappears in the Inactive List at the same position. An ephemeral self-exit is `delete`d from `this.sessions` (gone). A user-stopped (Stop/Restart precursor, `userStopped=true` → `'stopped'`) is NOT a self-exit and stays in the live map so `restart()` can respawn under the same logicalId. The two remaining Plan-01 scaffolds are GREEN.
