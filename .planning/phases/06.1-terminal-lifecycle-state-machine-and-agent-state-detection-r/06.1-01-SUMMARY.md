- **`❯` escape for the caret in `agent-state.ts`:** the numbered-menu regex needs the caret as an optional, non-decisive item prefix (`[❯>]?`), but the acceptance grep demands zero literal `❯` in the file. **2. [Rule 3 - Blocking / Plan-fact correction] The 002 capture has NO raw PTY data frames to replay**
- **Found during:** Task 2 (replay oracle)
- **Issue:** The plan's Task 2 instruction is to "replay the captured PTY data frames through `term.write`" and run `classify()` on settle. But `capture-claude.jsonl` is a FORENSIC log written by `record.cjs` — its events are `spawn`/`tick`(hash)/`state`/`settle`/`exit` (verified counts: {spawn:1, tick:513, state:11, settle:28, exit:1}). There are NO raw byte/data frames in the file, so feeding `@xterm/headless` raw PTY bytes is impossible. Each `settle` at `threshold===400` (11 of them) carries only the settled active line (`last`) and the per-signal booleans (`sig`), not the full region.
- **Fix:** Built the oracle the faithful way the spike's own `reanalyze.cjs` proved the finding: read the 11 `threshold===400` settle events, reconstruct each settled viewport frame from its recorded `last` + `sig` (a Claude-shaped numbered menu when `numbered_menu` fired + the recorded active line), write that frame into a real `@xterm/headless` Terminal (pinned 5.5.0, byte-identical viewport), read it back via the production `viewportLines()` shape, and run the shipping `classify()`. Deterministic — awaits xterm's write callback (post-parse), no timers. Result: 11 settles → exactly 1 WAITING, matching `reanalyze.cjs`.
- **Files modified:** src/shared/__tests__/agent-state-replay.test.ts, package.json
- **Verification:** `npm run test:unit -- agent-state-replay.test.ts` → 4 passed; the oracle asserts `waitingCount===1`, `freeCount===10`, and that the WAITING settle is the @29s confirmation; cross-checked against `node reanalyze.cjs` (11 settles → 1 WAITING).
- **Committed in:** db195ae (Task 2 commit)
