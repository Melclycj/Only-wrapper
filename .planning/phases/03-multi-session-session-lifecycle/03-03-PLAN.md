<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (contextBridge) | Stop/restart controls call the `ptyStop`/`ptyRestart` bridge methods with a renderer-supplied `id`; the startup-command E2E calls the existing `ptyCreate` contract directly. Still the only renderer↔main seam; no new bridge surface this plan. |
| main → child process | Restart re-spawns a real child; the in-flight grace timer + the old child must be reaped before/at respawn (orchestrated in main from 03-01). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / DoS | `ptyStop`/`ptyRestart` called with a forged/unknown `id` | mitigate | Main validates `id` against the live record store and ignores unknown ids (03-01 guard); the renderer only ever passes ids from its own session list. |
| T-03-03 | Injection | `startupCommand` exercised via the direct-`ptyCreate` E2E this plan | accept | Written as the user's own keystrokes into the user's own shell — no app-side eval/exec; runs with exactly the user's privileges (Security V5, carried from 03-01). |
| T-03-04 | Information Disclosure | startup command text could be logged when surfaced in the UI | mitigate | The renderer renders it into the user's own terminal only; no logging of the command text (carried from 03-01 V7). |
| T-03-05 | DoS / resource leak | Restart leaves the old child or the grace timer orphaned | mitigate | Restart orchestrates stop → await exit → respawn in main (one live pty per id); `disposeAll()` clears any in-flight `killTimer` (03-01). |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages (RESEARCH §Package Legitimacy Audit); no install task. |

No HIGH-severity threat is left without a mitigation.
</threat_model>
