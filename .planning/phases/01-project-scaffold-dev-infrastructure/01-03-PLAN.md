## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer (Chromium, untrusted web context) → main (Node.js, full privilege) | The Phase-1 attack surface (ASVS V4 Access Control). Crossed ONLY via contextBridge + ipcRenderer.invoke. |
| untrusted page / preload injection → preload script | Sandboxed preload limits available modules |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Elevation of Privilege | renderer accessing Node/Electron APIs directly | mitigate | buildWebPreferences sets nodeIntegration:false + contextIsolation:true + sandbox:true (D-07); security.guard.test asserts these values |
| T-1-04 | Elevation of Privilege | raw ipcRenderer exposed on window | mitigate | contextBridge.exposeInMainWorld('api', {getVersion}) is the only bridge (SC3); renderer imports only window.api; D-06 ESLint ban blocks electron/ipcRenderer imports under src/renderer/** (verified by `eslint src/renderer/`) |
| T-1-05 | Spoofing | preload script injection from untrusted URL | mitigate | sandbox:true restricts the preload to whitelisted modules (contextBridge, ipcRenderer) — no fs/child_process/npm reachable (Pattern 2 sandbox note) |
| T-1-IPC | Tampering | malformed args to api:get-version handler | accept | Phase 1 handler takes no user-supplied arguments (returns app.getVersion()); IPC argument validation begins in Phase 2 when PTY channels carry user input (RESEARCH Security Domain) |
