<objective>
Implement the shared identity contract that permanently separates logical session identity from the PTY process ID at the type level. Create src/shared/types.ts with the branded LogicalId type (D-04), the SessionStatus string-literal union (D-02), the SessionIconSpec discriminated union (D-03), and the full SessionRecord interface (D-01); create src/shared/id-factory.ts with the newLogicalId() factory wrapping uuid v4 (D-04); and create src/shared/api-types.ts as the pure-type window.api surface the renderer imports without touching electron.

This is Wave 2 of the Walking Skeleton. It turns the Plan 01 identity guard stub GREEN (IDENT-01, IDENT-02, D-05) and supplies the SessionRecord contract that Phases 2-5 build PTY, lifecycle, sidebar, and persistence against. The api-types.ts type module is the contract Plan 03's preload + renderer round-trip is checked against.

Purpose: This is the "can never be violated later" invariant. A branded LogicalId means a bare string or a stringified PID is a compile-time error wherever a LogicalId is required, so session maps/lookups are structurally forced to key on identity, never the process (IDENT-02).
Output: The stable shared TypeScript contract (types, id factory, api-types) consumed by every later phase.
</objective>
